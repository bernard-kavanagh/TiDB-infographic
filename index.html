The Disaggregated Architecture

TiDB's power comes from separating compute from storage. This allows for independent scaling of resources, providing unparalleled flexibility and efficiency for any workload.

The HTAP Advantage: One Database, Two Workloads

Hybrid Transactional/Analytical Processing (HTAP) means TiDB can serve as both your primary OLTP database for user-facing applications and your OLAP database for real-time analytics, eliminating complex and slow ETL processes.

TiDB vs. Traditional Databases

How does TiDB stack up against the databases you use today? It combines the best of both worlds: the familiar SQL interface of traditional RDBMS with the scalability of NoSQL.

Capability TiDB Traditional SQL (e.g., MySQL)
Scalability Model Horizontal (Scale Out) Vertical (Scale Up)
Consistency Strong (ACID via Raft) Strong (Single Node ACID)
Analytical Queries High-Performance (Columnar) Slow (Row-based, impacts OLTP)
Fault Tolerance Built-in, Automated Failover Requires manual setup (e.g., Replication)
Sharding Automatic & Transparent Manual & Complex Application Logic
